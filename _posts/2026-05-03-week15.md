## Learning from others

From watching other groups' presentations, I realized that they encountered very similar issues with our group trying to contribute to their respective open source projects. It seems like a consensus that on bigger projects, navigating the code base and finding an issue to work on can be really difficult. The "good first issue" tag are almost always useless since either it's too easy and there's ten new contributors wanting to work on it within the first minute or it's too hard and not really suited for a newcomer.

## Finishing up contributions to Pandas

I've also redone one of my bug [PRs](https://github.com/pandas-dev/pandas/pull/65117) because the maintainers intentionally want to keep pandas' core casting strict to prevent silent data corruption with unitless timedelta64 values. Instead of relaxing those global rules, I rewrote the fix to intercept and correct the unit at the pyarrow I/O layer before it attempts the invalid cast.
